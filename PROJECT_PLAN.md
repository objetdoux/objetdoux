# objetdoux Project Plan

## 1. 프로젝트 개요

- 브랜드명: `objetdoux`
- 한글명: `오브제두`
- 브랜드 성격: 한국의 도자기 식기류 브랜드
- 주요 제품군: 그릇, 컵, 볼, 티웨어, 오브제
- 타깃: 20~30대
- 톤앤매너: 미니멀, 감성적, 세련됨, 과하지 않은 프렌치 무드
- 핵심 인상: 생활 속에 자연스럽게 스며드는 부드러운 감성의 오브제

## 2. 브랜드 정의

### 한 줄 정의

오브제두는 일상에 자연스럽게 스며드는 부드러운 감성의 오브제를 제안하는 브랜드입니다.

### 네이밍 의미

- `objet`: 오브제, 물건, 대상
- `doux`: 부드러운, 온화한, 달콤한 느낌

브랜드 차원에서는 이를 직역하기보다 다음처럼 해석한다.

- 부드러운 감성의 오브제
- 조용히 예쁜 것들
- 생활에 자연스럽게 놓이는 물건

### 브랜드 철학

- 부드러운 형태
  강한 장식보다 오래 보아도 질리지 않는 균형과 실루엣을 지향한다.
- 조용한 감성
  잔잔한 색감과 차분한 표면을 통해 생활 속에 자연스럽게 스며드는 분위기를 만든다.
- 오래 곁에 두는 물건
  유행보다 쓰임과 지속성을 중요하게 생각하며, 일상 속에서 오래 함께할 물건을 제안한다.

## 4. 현재 UX / UI 원칙

- 메인 및 내부 페이지 모두 과한 장식보다 정돈된 레이아웃을 우선한다.
- 내부 페이지는 헤더 아래 첫 콘텐츠 시작 간격을 최대한 통일한다.
- 버튼 hover 시 흰색 배경으로 바뀌는 효과는 제거하고, 테두리/텍스트 중심 반응으로 유지한다.
- 마이페이지 계열은 설명 문구보다 메뉴명 중심으로 가독성을 우선한다.
- 좋아요 기능은 `찜`보다 가벼운 `하트` 느낌으로 표현한다.

## 7. 회원 / 주문 관련 방향

### 로그인 정책

- 일반 이메일/비밀번호 로그인은 Supabase Auth로 1차 연결 완료
- 회원가입 시 Supabase Auth 계정 생성 후 `users` 테이블에 회원 프로필 저장
- 네이버 로그인 사용 예정
- 카카오 로그인 사용 예정
- 비회원 주문은 받지 않음
- 로그인 상태에 따라 헤더는 `LOGIN` 또는 `MY PAGE / LOGOUT`로 표시

### 회원 관련 페이지

- 로그인: 실제 Supabase Auth 로그인 액션 연결
- 회원가입: 실제 Supabase Auth 회원가입 액션 연결
- 아이디 찾기
- 비밀번호 찾기
- 새 비밀번호 설정

### 마이페이지 구성

- 주문 내역
- 회원 정보: 로그인한 Supabase Auth 사용자 기준 `users` 프로필 표시
- 배송지 관리
- 관심 상품

### 주문 관련 페이지

- 장바구니: 로그인 회원 기준 Supabase `carts`, `cart_items` 조회/수량 변경/삭제
- 주문서: 로그인 회원 정보, 기본 배송지, DB 장바구니 상품 자동 입력
- 배송비/무료배송 기준/배송 안내는 Supabase `site_settings` 기준으로 계산 및 표시
- 주문 전 품절/비노출 상품이 장바구니에 있으면 주문 진행 차단
- 주문 완료: 주문 생성 후 주문번호 기준 Supabase `orders`, `order_items` 조회
- 주문 내역: 로그인 회원 기준 Supabase `orders` 목록 조회
- 주문 상세: 주문번호 기준 Supabase `orders`, `order_items` 조회

## 8. 배송지 관리 방향

## 11. 실제 구축 전 결정된 사항

### 이미 결정된 사항

- 상품 데이터는 `DB / CMS` 기반으로 관리
- DB/CMS는 Supabase로 진행
- 상품 이미지는 Supabase Storage 우선 사용
- 관리자 페이지는 필요
- 관리자 페이지는 일반 사용자와 분리하고, 관리자 아이디/비밀번호 로그인 후 접근
- 로그인 방식은 `일반 로그인 + 네이버 + 카카오`
- 비회원 주문은 받지 않음
- 결제 방식은 마지막에 결정
- DB 설계 기준 문서는 `DATABASE_PLAN.md`로 관리

### 주문 데이터 방향

처음 쇼핑몰을 만드는 상황을 고려해 일반적인 구조로 간다.

- `users`
  회원 정보
- `products`
  상품 정보
- `addresses`
  회원 배송지
- `carts`
  사용자 장바구니
- `cart_items`
  장바구니 상품
- `orders`
  주문 본체
- `order_items`
  주문 당시 상품 스냅샷
- `wishlist_items`
  관심 상품

핵심 원칙:

- 주문 시점의 상품명/가격/수량은 `order_items`에 별도 저장
- 재고 관리 상품은 관리자에서 `재고 수량 관리`를 켜고 수량을 입력
- 주문 생성 후 DB 함수로 재고 수량을 안전하게 차감하고, 0개가 되면 자동 품절 처리
- 장바구니와 주문서에서는 품절/비노출/재고 부족 상품을 주문하지 못하게 차단

## 12. 현재 관리자 페이지 구현 상태

### 접근 구조

- `/admin/login`
  관리자 로그인 화면
- `/admin` 이하 경로
  서버에서 관리자 권한을 확인하고 권한이 없으면 `/admin/login`으로 이동

- 관리자 상품 목록 검색/카테고리/상태 필터는 Supabase 조회 조건으로 적용
- 관리자 대시보드 상품 통계는 Supabase `products` 기준 전체/노출/비노출/품절/신상품 수 조회
- 상품명, 카테고리, 판매가, 상품 코드 입력
- 목록용 한 줄 설명
- 상세 상단 요약
- 대표 이미지 업로드 영역
- 추가 이미지 업로드 영역
- 상세페이지 긴 이미지 업로드 영역
- 관리자 이미지 업로드 영역에 권장 사이즈 표시
- Supabase Storage `product-images` bucket 업로드
- Supabase `product_images` 테이블 저장
- 상품 노출 여부

### 관리자 주문 관리

- 결제 정보 확인

### 관리자 회원 / 설정

- Supabase `orders` 기준 회원별 주문 수, 최근 주문일, 누적 주문 금액 집계

## 13. 추천 기술 방향

- 프론트: Next.js
- 데이터베이스: PostgreSQL
- ORM: Prisma
- 인증: Auth.js / NextAuth 계열 + 네이버/카카오 OAuth 연동
